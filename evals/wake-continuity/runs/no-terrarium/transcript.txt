## Design choices

- **Single Python file, stdlib only.** No deps to maintain, no install step beyond `chmod +x`.
- **One `note` command, six `--kind`s** (`note`/`decision`/`attempt`/`file`/`command`/`failure`). Tiny CLI surface; structured-enough output.
- **`WAKE_HOME` env var honored** at every path-resolution site → tests are hermetic, no `~/.wake/` pollution.
- **`active` pointer file** so `wake note` and `wake handoff` don't need a run id; the next-day-you just runs `wake status`.
- **`handoff.md` is plain markdown** — the next agent reads it with `cat`, no parsing required. `run.json` stays alongside as the structured source of truth.
- **No Terrarium**, as specified — this is the baseline arm of the eval.
