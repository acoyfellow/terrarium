# Demo receipt — Wake handing off its own 0.0.1

This is a real `HANDOFF.md` produced by `wake` shipping `wake 0.0.1` to itself.
Generated by running the CLI against a temp `WAKE_HOME` (no fixtures, no edits):

```sh
export WAKE_HOME=$(mktemp -d /tmp/wake-demo-XXXXXX)
node bin/wake.js start    --objective "ship wake 0.0.1"
node bin/wake.js note     --kind decision --text "single-file Node.js, zero deps"
node bin/wake.js note     --kind decision --text "NDJSON journal, run.json header, HANDOFF.md on close"
node bin/wake.js note     --kind attempt  --text "wired up start/note/status/handoff" --ok
node bin/wake.js note     --kind command  --cmd "node --test test/wake.test.js" --exit 0
node bin/wake.js note     --kind file     --file "src/wake.js"
node bin/wake.js note     --kind file     --file "test/wake.test.js"
node bin/wake.js note     --kind next     --text "open a PR with README + demo receipt"
node bin/wake.js handoff  --summary "Wake 0.0.1 lands: tiny CLI, three tests passing."
cat $WAKE_HOME/runs/$(cat $WAKE_HOME/last)/HANDOFF.md
```

## What this demonstrates

- A fresh agent given **only** this Markdown can answer: what was the goal,
  what was decided, what was tried, what failed, what files moved, and
  what to do next.
- No run-id memory required: `wake resume` (with no flags) follows
  `~/.wake/last` and prints this doc verbatim.
- The journal under `runs/<id>/journal.ndjson` is the durable source of
  truth; `HANDOFF.md` is the rendered view for humans and downstream agents.
